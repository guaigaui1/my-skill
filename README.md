# fable-learning · 寓言式讲解

*[English version](./README.en.md)*

一个 Claude Code Skill。用一个寓言间接讲透一个概念/设计理念/知识的内核,临近结尾才让人隐约悟到它是什么,故事之后再补一段点题解释。让读者在故事里亲历这个机制的因果,自己先"啊"一下,再点题。

技能正文见 [`SKILL.md`](./SKILL.md)。

## 安装

把整个目录(含 `SKILL.md`)放到 Claude Code 能发现 skill 的位置:

```
~/.claude/skills/fable-learning/SKILL.md      # 个人全局
# 或
<项目>/.claude/skills/fable-learning/SKILL.md  # 项目级
```

放好后,新开会话即可被自动发现。

## 使用方法

直接用自然语言触发即可,例如:

- "用寓言给我讲讲幂等性。"
- "幂等、重试、补偿这几个概念有什么区别和联系?打个比方。"
- "我搞不懂为什么要写单元测试,讲个故事让我懂。"

Claude 会按 `SKILL.md` 的三步来写:**先抓内核 → 寓言正文(不出现术语)→ 用 `---` 隔开点题**。

## 示例

下面是按本 skill 写出的一个完整例子。讲的概念是**幂等性 (idempotency)**,内核只有一个:**同一个操作做一次和做很多次,结果一样,所以重试是安全的。**

---

老周等电梯。

按钮按下去,亮了。他等了两秒,没动静,又按。还是没动静。他烦了,啪啪啪连按七八下,像要把按钮按进墙里。

旁边小孩看他,也跟着伸手去戳,戳得更欢。

电梯门"叮"地开了。

就一部电梯下来。不是七八部,不是十几部。老周按了十下,小孩按了二十下,该来的还是这一部,不多不少。两人挤进去,老周忽然觉得自己刚才那通乱按有点好笑——**按一下和按三十下,对电梯来说是同一件事**:你要它来,它就来一趟,你喊多少遍都是这一趟。

他想起昨天网购点"下单",页面卡住,他又点了一下,结果收到两件一模一样的货。那个按钮,可不像这个电梯按钮这么省心。

---

刚才那个电梯按钮,就是**幂等**的。

幂等性说的是:**一个操作,执行一次和执行 N 次,系统最终状态完全相同**。"召唤电梯"是幂等的——按几下都只来一趟;而那个"下单"按钮不是幂等的,点两次就多买一件,这就是老周的麻烦。

把故事元素对回去:**乱按按钮 = 客户端重试**(网络超时、用户不耐烦);**只来一部电梯 = 重复请求被收敛成一个结果**;**多买一件货 = 非幂等接口在重试下产生的副作用**。

它为什么这样设计:在分布式系统里,请求超时、丢包、重发是常态,你**没法阻止**重试发生。所以与其祈祷别人只点一次,不如把操作设计成"点几次都一样"——这样重试就从风险变成了免费的保险。读、查询天生幂等;而"创建订单""转账"这类带副作用的操作,得靠唯一请求 ID、去重表这些手段,人为把它变成像电梯按钮一样的东西。

用在哪:任何可能被重试的写操作(支付、下单、消息消费)。不用在哪:本就该累加的场景——比如"给我的账户充 10 块",你按一下是充一次,这时候要的恰恰不是幂等。
